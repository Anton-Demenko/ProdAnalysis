# MU-55-2024 (адаптация) — матрица трассируемости

Этот документ используется как формальная трассируемость «МУ-55-2024 → реализованные функции системы».
Номера MU55-PA-xx — внутренние идентификаторы требований, применяемые для защиты и демонстрации соответствия методике.

## MU55-PA-01 Почасовой производственный анализ
**Смысл:** фиксация и анализ выполнения плана по часам в рамках смены.  
**Реализация:** ProductionDay (08:00–16:00), HourlyRecord (8 часов, HourIndex 0..7), экран деталей смены.

## MU55-PA-02 План/факт по часу и по смене
**Смысл:** плановые и фактические показатели должны быть сопоставимы и рассчитываться автоматически.  
**Реализация:** HourlyRecord.PlanQty, HourlyRecord.ActualQty, итоги по смене на странице деталей.

## MU55-PA-03 Накопительный итог
**Смысл:** расчёт накопительных показателей по смене (CumPlan, CumActual, CumDeviation).  
**Реализация:** вычисления в DTO/странице смены (накопительные суммы по часам).

## MU55-PA-04 Отклонения и реакции
**Смысл:** если факт ниже плана — событие отклонения, фиксация реакции и закрытия.  
**Реализация:** DeviationEvent (Open/Ack/Closed), ACK/Close в UI, автозакрытие при выполнении плана, журнал EscalationLog.

## MU55-PA-05 Учет причин отклонений (простои)
**Смысл:** фиксация причин потерь времени с контролем корректности ввода.  
**Реализация:** DowntimeReason, HourlyDowntime (минуты), правило суммы минут ≤ 60 в UI.

## MU55-PA-06 Pareto-анализ
**Смысл:** выделение ключевых причин потерь по принципу Парето.  
**Реализация:** отчёт Pareto по простоям (таблица + доля + накопительная доля), экспорт CSV.

## MU55-PA-07 Интеграции/обмен
**Смысл:** возможность выгрузки/загрузки данных для внешних систем (Excel/ERP/MES).  
**Реализация:** CSV export/import (hourly), export pareto, export summary.

## MU55-PA-08 Цепочка помощи (Help Chain)
**Смысл:** фиксировать отклонения и обеспечивать эскалацию, если реакция отсутствует.  
**Реализация:** уровни L1/L2, фоновая эскалация (DeviationEscalationWorker), настройки через appsettings.

## MU55-PA-09 Доступность с ПК и мобильных устройств
**Смысл:** интерфейс доступен на разных устройствах.  
**Реализация:** Blazor Server UI, адаптивность — этап развития (см. дорожную карту, эпик H).

---

## Примечание о методике
В MVP используется адаптированная логика производственного анализа: почасовой контроль плана/факта, отклонения, причины потерь и цепочка помощи.
Документ служит для демонстрации связи требований и реализованных функций.
