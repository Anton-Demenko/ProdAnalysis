# ProdAnalysis MVP — соответствие требованиям (traceability)

## Критерии приёмки
1) Корректная реализация минимум одного способа анализа  
✅ Реализовано: анализ по такту (TaktSec), почасовой план/факт, накопительный итог.  
Поддерживается режим <1 изделия/час за счёт накопительного распределения PlanQty по часам.

2) Ввод и накопление данных в реальном времени  
✅ Почасовой ввод ActualQty + Comment.  
✅ Накопительные итоги на деталях смены, мгновенное пересчёт Dev.

3) Анализ причин отклонений и визуализация отчётов  
✅ Простои по причинам + ограничение ≤ 60 мин/час.  
✅ Отчёт Pareto по причинам простоев.  
✅ Сводка (plan/actual/deviation) по диапазону дат.

4) Интерфейс для ПК и мобильных устройств  
✅ Blazor Server, простая адаптивная вёрстка (flex + таблицы со скроллом).

5) Логическая связь данных, расчётов и требований МУ-55-2024  
✅ Структура данных: смена → часы → простои.  
✅ Правила: plan/actual по часам, отклонение при Actual < Plan, цепочка помощи.  
✅ Интеграции: CSV import/export как минимальный аналог Excel/ERP.

## Что сделано для «цепочки помощи»
- DeviationEvent создаётся при Actual < PlanQty, закрывается при Actual ≥ PlanQty
- ACK фиксируется пользователем
- Эскалация до L2 без ACK по таймеру (настраивается опциями)
- EscalationLog хранит историю действий и эскалаций
